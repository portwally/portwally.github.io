Privacy Policy

Last updated: 22 April 2026

1. Summary

Glean runs entirely on your Mac. It does not have a server, does not have an account system, and does not transmit any information about you or the files it scans. There is nothing to opt out of because nothing is collected.

2. What Glean reads from your Mac

To find recoverable disk space, Glean reads the file system at well-known locations used by supported AI apps — for example ~/.claude/, ~/Library/Application Support/Claude/, ~/Library/Application Support/Code/, ~/.gemini/, ~/.ollama/, and similar. It records the file path, display name, category, size, and safety level — all locally, in memory, for the duration of the session.

Glean never opens, reads, or transmits the contents of any file it finds. It only uses file-system metadata (path, size, existence).

3. What Glean does not collect

  • No personally identifying information
  • No account or login data — there is no account system
  • No analytics, telemetry, crash reports or usage tracking
  • No advertising identifiers
  • No network requests to any Glean servers — because there are no Glean servers

4. What happens when you clean files

When you confirm a cleanup, Glean calls the macOS API FileManager.trashItem(at:) for each selected item. The item is moved to your Trash — it is not permanently deleted by Glean. You can restore anything from the Trash until you choose to empty it.

Items that are flagged as "unsafe" (user settings, OAuth tokens, plugin configs) cannot be selected in the UI and are never moved.

5. Third-party services

Glean uses only Apple system frameworks (SwiftUI, Foundation, AppKit). It does not integrate any third-party SDKs, analytics, or advertising services.

6. Children's privacy

Glean is a general-audience developer tool. It does not knowingly process data from children under 13.

7. Changes

If this policy changes in a future version, the updated date at the top will reflect it. Because no information is stored by us, there is nothing to migrate or delete.
